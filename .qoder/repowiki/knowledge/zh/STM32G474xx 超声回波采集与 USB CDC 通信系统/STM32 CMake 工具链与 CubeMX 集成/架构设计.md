本模块由三个独立的 CMake 构件组成，均通过 `CMAKE_TOOLCHAIN_FILE` 引入：
- `gcc-arm-none-eabi.cmake`：基于 GNU ARM Embedded Toolchain 的工具链文件，固定 `arm-none-eabi-` 前缀、Cortex-M4 + FPV4-Single-Precision 浮点、硬浮点 ABI，链接器脚本硬编码为 `${CMAKE_SOURCE_DIR}/STM32G474XX_FLASH.ld`，启用 `-Wl,--gc-sections` 与 `--print-memory-usage`。
- `starm-clang.cmake`：STMicroelectronics ARM Clang 工具链，通过 `STARM_TOOLCHAIN_CONFIG` 在 STARM_HYBRID / STARM_NEWLIB / STARM_PICOLIBC 三种后端间切换，分别注入不同的 multilib 配置与启动对象（`-lcrt0-nosys` / `-lcrt0-hosted -z norelro`）。
- `stm32cubemx/CMakeLists.txt`：面向 STM32CubeMX 生成目录的顶层构建入口，要求 CMake ≥ 3.22，将 HAL/LL、USB CDC 中间件与应用源码分别组织为 OBJECT 库 `STM32_Drivers`、`USB_Device_Library`，并通过 INTERFACE 库 `stm32cubemx` 暴露包含路径与宏定义；最终链接到 `${CMAKE_PROJECT_NAME}` 可执行目标。
依赖方向：应用层 `CMakeLists.txt` → `stm32cubemx/CMakeLists.txt` → 两个工具链文件；工具链之间互不依赖，仅通过 `TOOLCHAIN_LINK_LIBRARIES` 变量向调用方回传需链接的数学库。
