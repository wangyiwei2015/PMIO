模块按 CubeMX 标准目录分层组织：
- `App/`：应用层组合与 CDC 类适配。`usb_device.c` 的 `MX_USB_Device_Init()` 是入口，依次调用 `USBD_Init`、`USBD_RegisterClass(&USBD_CDC)`、`USBD_CDC_RegisterInterface(&USBD_Interface_fops_FS)`、`USBD_Start`；`usbd_cdc_if.c` 实现 `USBD_CDC_ItfTypeDef` 函数表（Init/DeInit/Control/Receive/TransmitCplt），并通过全局 `UserRxBufferFS`/`UserTxBufferFS` 与上层交换数据；`usbd_desc.c/.h` 提供设备/配置/字符串描述符。
- `Target/`：硬件抽象层，`usbd_conf.c` 实现 HAL PCD 回调（Setup/DataIn/DataOut/SOF/Reset/Suspend/Resume/Connect/Disconnect/LPM）并桥接到 `USBD_LL_*` 底层接口，同时完成 USB IP 时钟（HSI48）、NVIC（USB_LP_IRQn, priority 0）、PMA 端点缓冲分配（EP0/EP1/EP2）。`USBD_LL_Init` 中通过 `HAL_PCDEx_PMAConfig` 静态分配 EP0(0x18)、EP1_IN(0xC0)/OUT(0x110)、EP2_IN(0x100) 等缓冲区地址。
依赖方向：`App` → USBD 核心 + CDC 类 → `Target` 的 `USBD_LL_*` / HAL PCD 回调 → HAL/LL 寄存器访问；CDC 类与应用之间仅通过 `USBD_CDC_SetTxBuffer/ReceivePacket/CDC_Transmit_FS` 等 API 解耦。
